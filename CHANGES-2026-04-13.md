## Código (`main.py`)

### Restrição de visita única parametrizável

**Motivação:** Decisão tomada em reunião com o professor — remover a restrição que limita o UAV a visitar cada sensor no máximo uma vez por rodada, para permitir revisitas e avaliar o impacto no desempenho.

**O que mudou:**

- `add_visit_constraints` recebe novo parâmetro `allow_revisit: bool = False`.
  - Quando `False` (padrão): comportamento original — `sum_t v[j,t] <= 1` é adicionada ao modelo.
  - Quando `True`: restrição removida — o UAV pode visitar o mesmo sensor várias vezes na mesma rodada.
- A definição de `y[j]` (sensor visitado ao menos uma vez) foi reescrita de uma equação exata para um par de desigualdades que funciona corretamente em ambos os modos:
  - `y[j] >= v[j,t]` para todo `t`
  - `y[j] <= sum_t v[j,t]`
- `build_optimization_model` recebe e repassa `allow_revisit` para `add_visit_constraints`.
- Novo argumento de linha de comando `--allow-revisit` (flag booleana, padrão desligado).

**Como usar:**

```bash
# Comportamento padrão (visita única — igual ao original)
python main.py

# Sem restrição de visita única
python main.py --allow-revisit

# Exemplo com cenário completo
python main.py \
  --sensors-csv setup/posicao/10/sensors_400x400.csv \
  --aoi-state /tmp/aoi_state.csv \
  --aoi-history /tmp/aoi_history.csv \
  --round-summary /tmp/round_summary.csv \
  --allow-revisit
```

## Experimentos e dados (`setup/anafi_usa/`)

### Conclusão dos experimentos Parrot Anafi USA
- Todos os 36 cenários (6 × `n_s` × 6 × `L`) foram executados com 30 rodadas cada.
- `BATTERY_MAX = 141_372 J` (capacidade real da bateria do Parrot Anafi USA).
- Cenário `P20-800` foi **apagado e refeito** (arquivo antigo continha 46 rodadas de duas execuções sobrepostas).
- Cenário `P30-400` teve `aoi_state` reconstruído a partir do round 23 do `aoi_history` e completado com 7 rodadas restantes. O arquivo final foi truncado para 30 rodadas.

### Correções de robustez no solver (`main.py`)
- **Limites de memória do Gurobi**: `Threads=2`, `NodefileStart=0.5 GB`, `SoftMemLimit=4 GB` — evita travamento/desligamento do computador em cenários grandes.
- **Tratamento de rodadas sem solução**: quando `m.SolCount == 0` (sem incumbente — por timeout, limite de memória ou inviabilidade), a rodada agora é registrada com zeros e a AoI é envelhecida normalmente, mantendo a cadeia de estado consistente.
