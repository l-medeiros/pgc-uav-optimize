## Código (`main.py`)

### Restrição de visita única parametrizável

**Motivação:** Decisão tomada em reunião com o professor — remover a restrição que limita o UAV a visitar cada sensor no máximo uma vez por rodada, para permitir revisitas e avaliar o impacto no desempenho.

**O que mudou:**

- `add_visit_constraints` recebe novo parâmetro `allow_revisit: bool = False`.
  - Quando `False` (padrão): comportamento original — `sum_t v[j,t] <= 1` é adicionada ao modelo.
  - Quando `True`: restrição removida — o UAV pode visitar o mesmo sensor várias vezes na mesma rodada.
- A definição de `y[j]` (sensor visitado ao menos uma vez) foi reescrita de uma equação exata para um par de desigualdades que funciona corretamente em ambos os modos:
  - `y[j] >= v[j,t]` para todo `t`
  - `y[j] <= sum_t v[j,t]`
- `build_optimization_model` recebe e repassa `allow_revisit` para `add_visit_constraints`.
- Novo argumento de linha de comando `--allow-revisit` (flag booleana, padrão desligado).

**Como usar:**

```bash
# Comportamento padrão (visita única — igual ao original)
python main.py

# Sem restrição de visita única
python main.py --allow-revisit

# Exemplo com cenário completo
python main.py \
  --sensors-csv setup/posicao/10/sensors_400x400.csv \
  --aoi-state /tmp/aoi_state.csv \
  --aoi-history /tmp/aoi_history.csv \
  --round-summary /tmp/round_summary.csv \
  --allow-revisit
```

## Código (`main.py`, `run_revisit_experiments.sh`)

### Limite de revisitas por sensor (`--max-revisits`)

**Motivação:** Experimentos com `--allow-revisit` causavam crash do computador (desligamento) mesmo com os limites de memória anteriores. Sem nenhum corte superior, o Gurobi explorava combinações onde o UAV visita o mesmo sensor dezenas de vezes, tornando a árvore B&B intratável mesmo para o menor cenário (5 sensores, 100 m).

**O que mudou:**

- `add_visit_constraints` recebe novo parâmetro `max_revisits: int = 1`.
  - Quando `allow_revisit=False`: comportamento original — `sum_t v[j,t] <= 1`.
  - Quando `allow_revisit=True`: impõe `sum_t v[j,t] <= max_revisits` em vez de remover o corte completamente. Devolve ao solver um limite superior forte que poda a árvore de busca sem proibir revisitas.
- `build_optimization_model` recebe e repassa `max_revisits`.
- Novo argumento de linha de comando `--max-revisits` (inteiro, padrão `3`).

### Redução de memória do solver (`solve_model`)

| Parâmetro | Antes | Depois | Motivo |
|-----------|-------|--------|--------|
| `Threads` | 2 | 1 | Menos nós B&B paralelos na RAM |
| `NodefileStart` | 0.5 GB | 0.1 GB | Começa a derramar para disco mais cedo |
| `SoftMemLimit` | 4 GB | 1.5 GB | Seguro para 8 GB RAM em dual boot |

### Script `run_revisit_experiments.sh`

- Passa `--max-revisits 3` explicitamente em todas as rodadas.

**Como usar:**

```bash
# Revisita com limite padrão (3 visitas por sensor)
python main.py --allow-revisit

# Revisita com limite customizado
python main.py --allow-revisit --max-revisits 2
```
